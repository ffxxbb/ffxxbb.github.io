2015总结

终于有时间静下来好好写一下总结了，总结2015，就是在无休止的加班中度过一年了。
不知不觉，已经踏入社会1年多了，一年多里，看着身边的小伙伴离职的离职，牛X的牛X，已经有小伙伴写的文章能上技术头条了，有小伙伴已经申请专利了，有小伙伴毕业出来创业已经能买房买车了，而自己还每天在为某个业务上的bug而奔波着，感觉自己已经跟小伙伴们的距离越来越远了。不管在技术还是在其他方面，总结2015一年就只有两个字：失败。

回顾2015

  • 1.搭建了这个博客，但只写了一篇搭建记录。
  • 2.看了前端黑客技术，python,java虚拟机，spring几本书，但是只停留在看的层面。
  • 3.在工作中对sql的写法，和代码的写法有了一定的进步，对公司业务较熟悉了，也自己完成一个新模块的开发。
  • 4.模仿别人的代码做了一个手机，电脑同步的小应用，但是很粗糙，没在写下去了。

展望2016

  • 1.学习基础知识，多线程，操作系统。
  • 2.读10技术的书籍，并且能精读4-5本，能用新的技术做一些小应用。
  • 3.做一个自己的项目。
  • 4.能坚持每个月写一下文章，再收拾一下这个博客，现在看起来还有些粗糙。
  • 5.整理和优化一下自己写过的代码。

希望2016年不要再这样朦朦胧胧的过下去了。

博客搭建记录

1.安装git客户端

下载msysgit,
Github账号与Git客户端需要有一个连接标识，即SSH密钥，通过这个密钥，我们可以在客户端下操作即可获取github服务器上的数据。这个密钥需要使用Git Bash来生成，操作如下：

  • 检查电脑上现有的ssh key:(在c:/users/administrator/.ssh)
    $cd ~/.ssh
    如果提示：no suchfile or directory 说明是第一次使用git
  • 生成新的ssh key
    $ ssh -keygen -t rsa -C "有箱地址@emai.com"
    输入命令后会提示保存的路径，直接回车，就是默认的路径
    然后提示输入密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。
