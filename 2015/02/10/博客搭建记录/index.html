博客搭建记录

1.安装git客户端

下载msysgit,
Github账号与Git客户端需要有一个连接标识，即SSH密钥，通过这个密钥，我们可以在客户端下操作即可获取github服务器上的数据。这个密钥需要使用Git Bash来生成，操作如下：

  • 检查电脑上现有的ssh key:(在c:/users/administrator/.ssh)
    $cd ~/.ssh
    如果提示：no suchfile or directory 说明是第一次使用git

  • 生成新的ssh key
    $ ssh -keygen -t rsa -C "有箱地址@emai.com"
    输入命令后会提示保存的路径，直接回车，就是默认的路径
    然后提示输入密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。
    如果看到以下页面，表示成功了，

  • 将生成的ssh key 复制到github
    经过以上操作后，会生成一个目录（.ssh），目录下有三个文件，其中SSH密钥在id_rsa.pub中
    然后在github页面输入ssh密钥,把id_ras.pub里面的秘钥复制过来就行

添加成功后，Github和Git客户端就拥有了连接认证标识，之后就可以通过Git客户端在本地进行操作了。
最后判断是否能连接github ssh –t git@github.com

  • 设置用户信息
    现在你已经可以通过SSH链接到GitHub了，还有一些个人信息需要完善的。
    Git会根据用户的名字和邮箱来记录提交。GitHub也是用这些信息来做权限的处理，输入下面的代码进行个人信息的设置，把名称和邮箱替换成你自己的，名字必须是你的真名，而不是GitHub的昵称。

2.在github上创建仓库

需要注意的一点：项目的名字是有规定的，必须是跟github用户名对应，用户名只ffxxbb,则项目名对应是ffxxbb.github.io

3.安装hexo

利用 npm 命令即可安装。（在任意位置点击鼠标右键，选择Git bash）
npm install -g hexo
创建hexo文件夹
安装完成，新建一个文件夹，用git bash执行以下命令, Hexo 即会自动在目标文件夹建立网站所需要的所有文件。
hexo init
安装依赖包
npm install
现在我们已经搭建起本地的hexo博客了，执行以下命令(在H:\hexo)，然后到浏览器输入localhost:4000就能看到

hexo generate
hexo server
可以简写：hexo g
          hexo s

好了，至此，本地博客已经搭建起来了，只是本地哦，别人看不到的。下面，我们要部署到Github。

4.部署

编辑_config.yml(在H:\hexo下)。(注意，每个属性，如type:,’:’后面必须空一格，不然会报错)

deploy: type: github repository: https://github.com/ffxxbb/ffxxbb.github.io.git branch: master
编辑完这样以后， 执行下列指令即可完成部署。

hexo generate
hexo deploy

执行命令是报错了
提示：[error] https://github.com/zhchnchn/zhchnchn.github.io is not a valid repositor URL!
_config.yml的配置错了，必须改成这样
deploy: type: github repository: git@github.com:ffxxbb/ffxxbb.github.io.git branch: master
